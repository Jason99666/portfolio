### 背景与目标

AI 领域信息爆炸式增长，**"信息生产"成本趋零，"信息筛选"成本却暴涨**。现有聚合工具（如 RSS 阅读器）只做收集不做判断，用户仍然需要逐篇阅读、自行筛选，深陷信息错失恐惧（FOMO）。

目标是构建一个具有"价值判断力"的资讯系统——通过多层评分引擎实现信息降噪，让用户在 5 分钟内获得当日核心洞察，而非此前手动浏览所需的 1-2 小时。

### 方案设计

核心设计决策是：不做"更多内容"，而是做"更好的筛选"。

系统的真正骨架是**三层知识架构**：

**第 1 层：单个信息层**——每篇文章独立评估，不依赖上下文。通过 7 维评分引擎（实体关联度 25%、新颖性 20%、深度 20%、可执行性 15%、时效性 10%、可信度 5%、紧迫性 5%）并行评分，输出综合分（0-100）和 Verdict 分级（T0 必读 ≥85 / T1 重要 / T2 参考 / T3 浏览 / T4 噪音）。高分文章额外触发四维战略评分（技术创新、影响深度、时效判断、学习价值），写入 StrategicInsight 深度表。

**第 2 层：事件聚合层**——同一事件的多篇报道自动聚类，避免重复阅读。基于 EntityBoard 构建实体看板（如 OpenAI、Anthropic、DeepSeek 等 24 个实体），按 7 天/30 天窗口统计：文章总量、平均评分、最高分、子主题分布。

**第 3 层：Wiki 知识层**——从资讯中沉淀长期知识结构。WikiEntity 追踪实体定位、技术路线图、里程碑与立场变化；WikiConcept 维护概念定义、演进历程与成熟度评估；WikiTrend 检测新兴趋势、衰退信号与预测。每月自动更新，由 WikiUpdateLog 记录审计信息（LLM 调用量、Token 消耗、成本）。

这三层从"单篇判断"→"事件聚合"→"知识沉淀"，让系统不仅是资讯过滤器，更是个人 AI 领域的动态知识库。

### 技术架构

```
微信公众号 / 开源项目 / RSS 订阅
    ↓
[第 1 层] 数据采集
    - WeWe Docker 抓取微信公众号内容
    - GitHub API + 多源聚合器汇总开源动态
    - Feed Parser 解析 RSS 订阅
    - 统一输出为标准化内容格式，存入内容池
    ↓
[第 2 层] 三层过滤漏斗
    - 白名单过滤：基于信源可信度快速放行或拦截
    - 轻量模型初筛：小模型判断内容相关性，控制成本
    - 七维度大模型精评：信息密度、创新性、影响力等深度评估
    ↓
[第 3 层] 知识架构
    ├─ 单个信息层：Event + 7维评分(并行7线程) + 四维战略评分 → Verdict分级(T0-T4)
    ├─ 事件聚合层：语义相似度聚类 → EntityBoard 实体看板(7天/30天窗口)
    └─ Wiki 层：WikiEntity(实体档案) + WikiConcept(概念演进) + WikiTrend(趋势追踪)
    ↓
[第 4 层] 展示
    - 每日 Top N 资讯面板
    - 实体看板视图
    - Wiki 知识图谱
    - Distilled 评分对比验证
```

### 业务结果

- **效率提升 12-24 倍**：5 分钟完成此前需 1-2 小时手动浏览的每日资讯聚合
- **API 成本降低 70%**：三层漏斗策略在保证质量的同时大幅削减大模型调用量
- **信息降噪**：七维度评分引擎 + 事件聚合有效过滤低质量内容，用户只需阅读每日 Top N 即可掌握核心动态
- **质量可验证**：205 个自动化测试用例保障各模块独立正确性

### 反思与迭代

争夺用户注意力的关键在于实现"主观萃取"——不是让系统代替你判断，而是把你的判断标准模式化。下一步方向是建立用户反馈层，将评分系统从"通用标准"进化为"个性化权重"，实现千人千面。
