### 背景与目标

VC 行业尽调质量高度依赖个人经验，新人培养周期长，普遍缺乏将"投资直觉"转化为结构化分析框架的沉淀机制。更根本的问题是：**投资直觉能否被拆解为可复现的分析框架？**

基于自身主导 8 个项目（累计 1.6 亿元）、担任 2 个项目委派董事的实战经验，目标是将隐性的投资判断力产品化，通过 Multi-Agent 架构提升团队整体尽调提问的深度与质量。

### 方案设计

关键决策是**聚焦尽调提问环节而非覆盖全流程**——"提什么问题"直接决定尽调深度，且边界清晰、适合用 LLM 驱动。

架构层面选择 **Multi-Agent（多智能体）架构**而非单一 Agent：尽调天然涉及多个专业维度（行业格局、团队能力、财务健康、合规风险），单一 Prompt 难以同时保证各领域的深度。拆分后每个 Agent 可独立维护领域知识库，且可以针对每个领域单独优化 prompt 和检索策略。

系统分为四层：

1. **Orchestrator（编排器）**：接收用户输入，分析任务意图，将请求分发给相关领域 Agent，汇总输出后生成结构化尽调问题清单
2. **4 个专业领域 Agent**：行业、团队、财务、风控，每个 Agent 独立维护领域知识
3. **RAG 知识库**：商业计划书库 + 访谈问题清单库 + 投资逻辑库，为 Agent 提供检索增强，确保输出有事实依据
4. **质量分层策略**：现有架构兜住下限（初阶同事也能产出结构化问题），未来通过复杂编排能力提高上限

### 技术架构

```
用户尽调需求（项目名 / 行业 / 特定关注点）
    ↓
[第 1 层] Orchestrator（意图编排）
    - FastAPI 接收请求，分析任务意图
    - 识别涉及的专业维度（行业/团队/财务/风控）
    - 并行分发至相关领域 Agent
    ↓
[第 2 层] 多 Agent 协同
    - 行业 Agent：市场格局、竞争态势、壁垒分析
    - 团队 Agent：创始人背景、组织能力、激励结构
    - 财务 Agent：商业模式、财务健康度、增长假设
    - 风控 Agent：合规风险、法律尽调、退出路径
    ↓
[第 3 层] RAG 检索增强
    - ChromaDB 向量检索：商业计划书、访谈问题清单、投资逻辑库
    - 为每个 Agent 提供事实依据，降低幻觉
    ↓
[第 4 层] 输出
    - 汇总各 Agent 输出，生成结构化尽调问题清单
    - 按优先级与领域分类，标注风险等级
    ↓
[持久层] PostgreSQL 存储对话历史与知识库元数据
```

### 业务结果

- **效率提升 60%**：尽调问题准备从手动整理（通常 4-6h ）缩短至 1 小时以内
- **质量兜底**：有效兜住了初阶同事的尽调质量下限——缺乏经验的新人也能产出结构化的问题清单，同时偶尔能产生超出预期的提问维度


### 反思与迭代

生产场景需求需足够聚焦，项目价值在于问题的深度而非广度。现有架构已经兜住下限，下一步是拓展复杂编排能力以提高上限。

这段经历也让我理解到：**VC 投资经理与 AI 产品经理在需求识别能力上高度同构**——两者都需要在信息不完整、环境快速变化的条件下，用结构化方法判断"什么是真需求"。
